## 索引原理

#### 1.b+树

```python
1.b是balance平衡的意思，为了保证每一个数据查找时经历的IO次数都相同

2.只在叶子节点存储数据，可以降低树的高度

3.叶子节点之前加入了双向连接，为了查找玩味的时候比较快
```

#### 2.聚集索引和非聚集索引

```python
1.聚集索引
全表的数据都存储在叶子节点上 —— Innodb存储引擎中的主键

2.非聚集索引
叶子节点不存放具体的整行数据，而是存储的这一行的主键的值
```

#### 3.正确的使用MySQL数据库

```python
1.从库的角度
	搭建集群
    读写分离
    分库
2.从表的角度
	合理安排表与表之间的关系：还拆的拆，该合的合
    把固定长度的字段放在前面
    尽量使用char而不是varchar
3.从操作数据的角度
	尽量在where字段就约束到一个比较小的范围
    尽量使用连表代替子查询
    删除数据和修改数据的时候条件尽量使用主键
    合理的常见和使用索引
    	没有命中索引
        1.查询的字段不是索引字段
        2.在条件中使用范围，结果的范围越大就越慢，范围越小就越快
        3.like 'a%'命中索引，like '%a'不能命中索引
        4.条件列不能参与计算，不能使用函数（就是where和having的后面）
        5.and\or
        	and条件相连，有一个有索引，就会命中索引
            or条件相连，所有的列都有索引才会命中索引
        6.联合索引
        	遵循最左前缀原则，企鹅冲出现范围开始索引失败（也就是查询条件中必须带着创建时处				于最左边的那个索引值
        7.条件中的数据类型和实际字段的类型必须一致
        8.select字段中应该包含order by中的字段
```
